# UX Task Card: State Persistence, Refresh Recovery, Multi-tab Safety

## Covers

- 31, 32
- 38, 39, 41
- 53, 55

## Problem (plain language)

用户输入很容易“白做”：
- Step3 的变量修正/答案没持久化，刷新/关闭就丢。
- 浏览器后退/前进会导致状态错乱。
- 多标签页打开同一个 job 会互相覆盖 localStorage。
- 确认后锁定导致用户看不到自己当时提交的内容，难以复核。

## Acceptance

- [ ] Step3 的变量修正与问题答案在本地持久化（刷新可恢复）
- [ ] 浏览器后退/前进不会造成“看起来回去了但状态没恢复”的错乱
- [ ] 多标签页冲突有提示（至少检测并告知“另一标签页正在编辑同一任务”）
- [ ] 确认锁定后仍可查看自己提交的修正/答案（只读）
- [ ] 提供轻量的撤销/回退机制（至少不必“全清空”才能改错）
