# UX Task Card: Upload, Preview, and Sheet Selection Workflow

## Covers

- 5, 26, 27
- 21, 22
- 35, 61
- 56

## Problem (plain language)

上传和预览是“核心入口”，但现在不够清晰、不够可控：
- 拖拽区缺少强提示与图标；错误格式/超大文件没有即时反馈。
- 文件名被截断但无法查看完整名称。
- 单元格截断固定 120 字符，用户无法展开查看完整内容。
- sheet 选择/记忆体验不足（尤其多表场景）。
- 预览表格内容不易复制。

## Acceptance

- [ ] 上传区明确提示“拖拽或点击上传”，展示格式与限制（大小/类型）
- [ ] 拖入不支持的文件：立即提示（不等到后端报错）
- [ ] 大文件策略明确（大小阈值提示 + 降级预览策略）
- [ ] 文件名截断可悬停看全称
- [ ] 单元格截断可展开（popover/modal），并解释截断规则
- [ ] sheet 选择器记住用户选择（刷新预览不丢）
- [ ] 表格内容与列名可复制（不需要开发者工具）

## Dependencies

- 若要支持“每个文件都选 sheet”，可能需要扩展前端数据结构与后端 preview 参数（需在后续实现 issue 中评估契约）。
